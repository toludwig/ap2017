Objective

The objective of this assignment is to gain hands-on programming experience with Haskell.

The goal is to implement a library, Curves, for working with piecewise linear curves in the plane (based on an idea by Hansen and Rischel, who in turn got the idea from Maarten M. Fokkinga).

A piecewise linear curve is a nonempty sequence of n points P0, P1, …, Pn − 1, connected by a sequence of linear segments: P0P1, P1P2, …, Pn − 2Pn − 1. We call P0 the starting point, and Pn − 1 the end point of the curve.

The library should implement the following types and functions:

  • The data type Point with two Double-typed components.

  • A function point:

    point :: (Double, Double) -> Point

    for construction a point from it’s x and y pair of coordinates.

  • Two extractor functions pointX and pointY:

    pointX :: Point -> Double
pointY :: Point -> Double

    such that

    p == point (pointX p, pointY p)

  • Due to the imprecision of floating point numbers, we decide to only care about the first two digits after the decimal point. This means that all points differing in x and y coordinate by less than 0.01 are equivalent.

    Declare Point to be an instance of Eq such that e.g. point (0,0) == point(0.009, 0.009) but point(0,0) /= point(0.01,0.01).

    Please note, this does not allow for more precise comparisons of points. Quite the contrary: point(5,5)==point(5.01,5) for any trivial implementation due to the way floating point works. The reason we ask you to do this is just so that you can test the below functions using trigonometric properties.

  • The type Curve for representing curves.

  • A function curve:

    curve :: Point -> [Point] -> Curve

    for constructing a curve consisting of the given starting point and subsequent sequence of points.

  • A function connect:

    connect :: Curve -> Curve -> Curve

    where connect c1 c2 is the curve starting with c1, a segment from the end point of c1 to the starting point of c2, and the curve c2.

  • A function rotate:

    rotate :: Curve -> Double -> Curve

    where rotate c d is the curve c rotated d degrees about the origin, (0,0), in a counter-clockwise direction. You may use the Haskell functions sin and cos, but note that these expect an angle in radians.

  • A function translate:

    translate :: Curve -> Point -> Curve

    where translate c p is the curve c translated in the plane. The distance and direction of the translation is determined by the vector from the starting point of c to the point p. p should therefore be (within floating point error of) the starting point of the translated curve.

  • The type Line and the function reflect:

    data Line = Vertical Double | Horizontal Double
reflect :: Curve -> Line -> Curve

    where reflect c $ Horizontal d is the curve c reflected in the horizontal line with y=d, and reflect c $ Vertical d is the curve c reflected in the vertical line with x=d. Reflection should not reverse the order of the points.

  • A function bbox:

    bbox :: Curve -> (Point, Point)

    that computes the lower left and the upper right corner of the bounding box for c. The curve should touch the bounding box on all sides.

  • The functions width and height:

    width :: Curve -> Double
height :: Curve -> Double

    which compute the width and the height of the bounding box of the curve.

  • A function toList:

    toList :: Curve -> [Point]

    that computes the list of curve points P0, P1, ..., Pn − 1.

  • A function normalize:

    normalize :: Curve -> Curve

    which moves the curve into the first quadrant, i.e., so that the lower left corner of the bounding box ends up at (0,0).

What to hand in

You should hand in two things:

  1. A short report, report.pdf, explaining your code, and containing an assessment of your implementation, including what this assessment is based upon. In the report, you should also:

    1. Discuss how you ensure that the user of your module cannot construct empty curves.

    2. Tell us how we can run your code and reproduce your test results.

  2. A ZIP archive src.zip, containing one directory src, containing your source code and tests. It should be possible to run your source as submitted.

To keep your TA happy, follow these simple rules:

  1. Clean up your code before you submit.
  2. Test your code before you submit.
  3. Comment the non-trivial parts of your code.
  4. Your module should not export partial functions.
  5. GHC, with the parameters -W set, should not yield any errors or warnings for your code.
  6. hlint should not yield any hints for your code.

Part 2: Generate SVG

Scalable Vector Graphics (SVG) is an XML-based 2D vector image format viewable in a modern internet browser. The image canvas is the first quadrant of a Cartesian coordinate system. The SVG coordinate system origin is the top-left corner of the image, and the first quadrant extends in a down- and right-wise direction. We choose to use pixels as the unit of measure.

  • Write a function toSVG:

    toSVG :: Curve -> String

    that generates an SVG string represnetation of the form:

    <svg xmlns="http://www.w3.org/2000/svg"
      width="...px" height="...px" version="1.1">
<g>
<line style="stroke-width: 2px; stroke: black; fill:white"
            x1="..." x2="..." y1="..." y2="..." />
...
</g>
</svg>

    The width and height, when used here, should be ceiled and integral. The x and y coordinates should use double digit precision. For instance, you may use the Haskell function printf with the arguments "%.0f" and "%.2f", respectively.

    For instance, both toSVG $ curve (point (0,10)) (map point [(5,0),(10,10),(0,10)]) and toSVG $ normalize $ curve (point (-5,15)) (map point [(0,5),(5,15),(-5,15)]) should produce:

    <svg xmlns="http://www.w3.org/2000/svg"
      width="10px" height="10px" version="1.1">
<g>
<line style="stroke-width: 2px; stroke: black; fill:white"
            x1="0.00" x2="5.00" y1="10.00" y2="0.00" />
<line style="stroke-width: 2px; stroke: black; fill:white"
            x1="5.00" x2="10.00" y1="0.00" y2="10.00" />
<line style="stroke-width: 2px; stroke: black; fill:white"
            x1="10.00" x2="0.00" y1="10.00" y2="10.00" />
</g>
</svg>

  • Write a function toFile

    toFile :: Curve -> FilePath -> IO ()

    that writes the SVG string representation of the curve to a file with the specified path. The function should overwrite the file if it already exists. Hint: Use the Haskell function writeFile. (FilePath is a type in the standard Haskell library.)

Part 3: Test with Hilbert curves

Test that you get the following figure:

by computing hilbert $ hilbert $ hilbert $ hilbert $ curve (point (0,0)) [], where hilbert is the following function:

hilbert :: Curve -> Curve
hilbert c = c0 `connect` c1 `connect` c2 `connect` c3
   where  w = width c
          h = height c
          p = 6

          ch = reflect c $ Vertical 0

          c0 = ch `rotate` (-90) `translate` point (w+p+w, h+p+h)
          c1 = c `translate` point (w+p+w, h)
          c2 = c
          c3 = ch `rotate` 90 `translate` point (0, h+p)
