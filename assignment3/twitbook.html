Objective

The objective of this assignment is to gain hands-on programming experience with Prolog, and in particular with declarative characterizations of positive and negative knowledge.

This assignment is about analyzing the social relations in a fictional social network called “TwitBook”.

A TwitBook friend graph consists of a set of people, where for each person, there is a set of other people that this person considers to be his or her friends. A friend graph is represented as a list of Prolog terms, each of the form person(X, [X_1, ..., X_n]), where X is an atom representing a person, and X1, …, Xn (where n ≥ 0) is that person’s friend list, in no particular order, without duplicates, and not including X itself. The list contains only one person by any given name X, and all members of the network have a (possibly empty) friend list.

For example, consider the following friend graph G1, where each node represents a person on TwitBook, and a directed edge from X to Y denotes that Y is on X’s friend list:

This graph can be represented with (for instance) the Prolog term:

[person(kara, [barry, clark]),
 person(bruce, [clark, oliver]),
 person(barry, [kara, oliver]),
 person(clark, [oliver, kara]),
 person(oliver, [kara])]

For easy experimentation, you can declare a Prolog fact:

g1([person(kara, [barry, clark]), person(bruce, [clark, oliver]), ...]).

Then the query ?- g1(G), pred(G, X, Y). will instantiate the variable G to the above representation of G1 for the second subgoal.

Predicates

In the following, you may assume that G and H are fully instantiated (i.e., ground), well-formed Prolog representations of friend graphs.

Warm-up: likes and dislikes

  1. We say that X likes Y in the graph, if Y is on X’s friend list (but not necessarily vice versa). For example, in G1, Bruce likes Clark.

    Write a predicate likes(G, X, Y) that is true whenever X likes Y in the graph G. For example ?- g1(G), likes(G, bruce, clark). should succeed, while the same query with the two name swapped should fail. A query where X and/or Y are uninstantiated variables should succeed repeatedly with all relevant pairs (and then fail).

  2. In general, many people in the network may not even be aware of each other, so not being on someone’s friend list is not in itself significant. However, we say that X dislikes Y if Y likes X, but X does not like Y back. Thus, in G1, we would say that Clark dislikes Bruce, while he neither likes nor dislikes Barry.

    Write a predicate dislikes(G, X, Y) that succeeds whenever X dislikes Y in the graph G. Hint: start by defining an auxiliary predicate different(G, X, Y) that succeeds whenever X and Y are different members of the network G.

Level 1: local connections

  1. We say that a network member X is popular if everyone whom X likes, likes him/her back. For example in G1, Kara is popular, while Clark is not.

    Write a predicate popular(G, X) that succeeds whenever X is popular in G.

  2. Conversely, a network member X is an outcast if everybody whom X likes, dislikes (i.e., fails to like back) him/her. For example, in G1 Bruce is an outcast.

    Write a predicate outcast(G, X) that succeeds whenever X is an outcast in G.

  3. A network member X is said to be friendly if X likes back everyone who likes him/her. For example, in G1, Barry is friendly.

    Write a predicate friendly(G, X) that succeeds whenever X is friendly in G.

  4. Conversely, a network member X is hostile if X dislikes everyone who likes him/her. For example, in G1, Oliver is hostile.

    Write a predicate hostile(G, X) that succeeds whenever X is hostile in G.

    Note that the first two predicates above consider how others behave towards X, while the last look at how X behaves towards others.

Level 2: global connections

  1. We say that X admires some other person Y if X likes someone who either is Y, or themselves admires Y – i.e., if there is a chain of likes from X to Y. For example, in G1, Bruce admires Kara, while the opposite is not true.

    Write a predicate admires(G, X, Y) that succeeds whenever X admires Y in G. Note that the graph G may contain cycles, which you need to detect in order to avoid going into an infinite recursion.

  2. We say that X is indifferent to some other person Y if X does not admire Y, in the sense above. Thus, for any pair of network members X and Y, exactly one of these conditions should hold: X admires Y, X is indifferent to Y, or X and Y are the same person.

    Write a predicate indifferent(G, X, Y) that succeeds whenever X is indifferent to Y in G.

Level 3: whole-graph properties

  1. Sometimes two friend graphs may describe identical connection patterns, except for the names of the people involved. We say that two social graphs G and H describe the same world if there is a one-to-one correspondence between the people in one and the other, such that everyone has exactly the same friends in the two graphs.     Write a predicate same_world(G, H, A) that succeeds whenever G and H describe the same social graph, and A is a list of pairs that says, for each person in G, who the corresponding person is in H. In the example above, with G and H taken as the Prolog representations of G1 and G2, A would be the list [(kara,supergirl),(bruce,batman),(barry,flash),(clark,superman),(oliver,green_arrow)].

  1. Optional! Write a predicate different_world(G, H) that succeeds if and only if the goal same_world(G, H, _) would fail.

Restrictions

Your predicates should work for both checking and enumerating solutions. The predicates should eventually fail (rather than loop forever), if the predicate does not hold of its arguments, or after finding all the solutions.

It is not a requirement that every solution be found exactly once, but you will probably find it easier to argue that your program is correct (i.e., allows neither too few nor too many solutions) if it only finds the same solution multiple times because it is a solution for different reasons.

For this assignment, efficiency is not a concern; that is, you need not worry about whether your programs would scale to much larger graphs than in the examples.

Important: Your program must consist of pure facts and rules only; do not use any built-in Prolog predicates or control operators – including in particular, but not limited to: equality/inequality (“=” and “\=”), arithmetic (“is”), disjunction (“;”), cuts (“!”), or negation-as-failure (“\+”). If you need any standard utility predicates, such as select/3, include their definitions in your program, but be sure to name them something different from the built-in ones.

Exception: For your testing, you are allowed to use all of Prolog and whatever extentions SWI Prolog offers.

To keep your TA happy, follow these simple rules:

  1. Clean up your code before you submit.

  2. SWI Prolog should not yield any errors or warnings for your code.

  3. You should comment your code properly, especially if you doubt the correctness or if you think there is a more elegant way to write a certain piece of code. A good comment should explain your ideas and provide insight that the code otherwise does not.

  4. Adhere to the restrictions set in the assignment text. Only use pure Prolog in your solution.
